詳細：
bzip2はブロックソート法（バロウズ-ホイラー変換）とMTF (Move-To-Front) 法、ハフマン符号化法を用いてファイルを圧縮します。従来のgzipやZIPといったデータ圧縮法と比べ、より高い圧縮率を誇っています。
オプション：
-c
圧縮、または標準形式の出力に復元する。
-d
強制展開する
-f
出力されたファイルを強制的に上書きする（標準では上書きされません。また、このオプションはファイルへのハードリンクを強制的に無効にします。）
-k
入力ファイルを削除せずに保管する。
-q
重要なメッセージのみ出力する。
-s
実行速度を犠牲にして、メモリーをセーブする
-z
強制圧縮する
例：
bzip2 -c fileone.txt filetwo.txt > output.bz2

cmpユーティリティーは２つのファイルを比較して、結果を標準形式で出力します。通常、ファイルに違いがない場合、cmpは何の結果も出力しません。ファイルの内容が異なる場合は、ファイルの最初の差異のバイト数とライン数を出力します。
例：
cmp file1 file2

詳細：
ディレクトリーの内容をリストとして出力する。
オプション：
-a
すべてのファイル（.から始まる、隠しファイルも含む）を表示する。
-A
隠しファイルも含めた、全てのファイルを表示します（ただし、現在のディレクトリー、または親ディレクトリーは表示しません。）
-b
出力不可能な文字列を8進法に基づいて、\dddとして強制的に出力する。
-c
iノード（ファイルの作成、モード変更など）が最後に編集された時間で、ソート（-tオプション）、または出力（-lオプション）する。
-C
エントリーが列ごとに並べられた、複数列形式での出力（標準のオプションになっています。）
-d
引数がディレクトリーの場合、ディレクトリーの中身ではなく、名前だけをリストとして出力する。
-f
各引数を強制的にディレクトリーとして認識し、各項目内の名前をリストとして出力する（このオプションは、-l、-t、-sと-rオプションを自動的に無効にし、-aを有効にします。出力順序は項目がディレクトリー内に表示される順になります。）
-F
ディレクトリはスラッシュ（/）、ドアは大なり記号（>）、実行可能ファイルはアスタリスク（*）、FIFOは縦線（|）、シンボリックリンクはアットマーク（@）、AF_Unixアドレスファミリーソケットは等号（=）でマークされます。
-g
所有者が出力されない以外は-lオプションと同様。
-i
各ファイルのIノード数をレポートの最初の列に出力する
-l
パーミッション、所有権、サイズ、最後の編集日時などの情報を表示します。
-L
引数がシンボリックリンクの場合、リンク自身ではなく、リンクが参照しているファイル、またはディレクトリーをリストとして出力する。
-m
ストリーム形式出力（ファイルはコンマで区切られて、ページを横断する形で表示されます。）
-n
所有者UIDと、グループGIDがテキストの代わりに出力される以外は、-lと同様。
-o
グループが出力されない以外は、-lと同様。
-p
全てのディレクトリーの前に、スラッシュ（/）を表示する。
-q
ファイル名の出力不可能な文字列を疑問符（？）として強制的に出力する。
-r
ファイルの表示順序を逆にする。
-R
サブディレクトリーの中身も含む。
-s
各エントリーのサイズをブロックで表示する（間接ブロックを含む）
-t
ファイルの編集時間を表示します。
-u
ファイルが最後にアクセスされた時間を使って、ソート（-tオプション）、または出力（-lオプション）する。
-x
行毎ではなく、列毎のエントリーのリスト
例：
ls -al
ls -lsR uploads

オプション：
-a

-d
DOSからUNIXへ変換する（プログラムはファイルを特定の形式から別の形式に変換します。例えば、プログラムがdos2unixの場合、通常、DOS形式をUNIX形式に変換します。）
-e

-f
通常、dos2unix、またunix2dosが書き込み権限のないファイルを見つけた場合、そのファイルは処理しません。このオプションはファイルが読み込み専用の場合でも、強制的に変換を行うようにします。
-o
バックアップを残さずに、元のファイルを上書きする（標準になっています。）
-p
ファイルの所有権や、時間をそのままにします。Linuxのようなシステム上では、ファイル所有権はユーザーがルート（管理者）の場合にのみそのままにされ、その他の場合は、時間のみが変更され、ファイルの所有権の変更はエラーを出すことなく失敗するようになっています。もし、ファイルの所有権が変更できない時にエラーを表示させたい場合は、-vオプションを使ってください。

DOSからUNIXへ変換する（詳細は上記の-dオプションを参照。）
-v
詳細モード。

オプション：
-b
出力された行に、ファイル名を追加しない（Briefモード）
-c
マジックファイルのパースされた形式を、内容にチェック用に出力します（通常、新しいマジックファイルをインストールする前にファイルをデバッグするために、-mオプションと一緒に使用されます。）

fileコマンドが、人間が読める形式の代わりにMIME形式文字列を出力するようにする

-r
出力不可能な文字列を\oooに変換しない（通常、出力不可能な文字列は8進法に基づいて変換されます）
-v
プログラムのバージョンを出力し、実行を終了する。
-z
圧縮ファイルの内部を参照する。

file *.txt
